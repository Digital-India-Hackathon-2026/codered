1. What is an LLM?

An LLM is a Transformer-based neural network trained to predict the next token in a sequence. By learning this task on massive amounts of text, it develops the ability to generate language, answer questions, summarize documents, and write code.

2. What is a token?

A token is the basic unit processed by an LLM. It may be a whole word, part of a word, or punctuation, depending on the tokenizer.

3. What are embeddings?

Embeddings are dense vector representations of tokens that capture semantic meaning. Tokens with similar meanings tend to have similar embeddings, allowing the model to understand relationships.

4. Explain self-attention.

Self-attention lets each token determine which other tokens are most relevant when creating its representation. This helps the model capture context and long-range dependencies. It's the same core idea used in Vision Transformers, except the inputs are text tokens instead of image patches.

5. What is the context window?

The context window is the maximum number of tokens the model can process at one time. Anything beyond that limit isn't directly considered unless it's summarized or retrieved again.

6. Why are Transformers better than RNNs?

Transformers process tokens in parallel using attention, making training much faster and allowing them to capture long-range dependencies more effectively than sequential RNNs.

7. What is temperature?

Temperature controls randomness during text generation. Lower values produce more predictable outputs, while higher values encourage more diverse and creative responses.

8. Why do LLMs hallucinate?

LLMs generate the most probable next token rather than verifying facts. If the training data is incomplete or the prompt lacks context, the model can produce confident but incorrect information.

9. What is the difference between training and inference?

During training, the model learns by updating its weights. During inference, the weights are fixed, and the model only generates predictions based on what it has already learned.

10. What is positional encoding?

Positional encoding provides information about token order because attention alone doesn't know the sequence of words.

Advantages of RAG

✅ Reduces hallucinations

✅ Uses private documents

✅ Handles frequently changing data

✅ No retraining

✅ Cheaper than fine-tuning

Limitations

❌ Poor retrieval → poor answer

❌ Bad chunking hurts accuracy

❌ More latency than calling an LLM directly

❌ Doesn't permanently teach the model new knowledge
